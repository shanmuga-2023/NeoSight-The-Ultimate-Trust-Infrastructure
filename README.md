## 🎯 Core Problem
Document forgery is rapidly evolving with sophisticated digital editing tools and AI manipulations. Current detection systems rely heavily on opaque "black-box" models that output probability scores without any explanation—rendering them completely unusable in real-world verification workflows like government processing, recruitment, and border control.

## 💡 Our Solution
**NeoSight** acts as a scalable trust infrastructure that not only detects forgery but *explains*, *proves*, and *builds trust*.

## 🛠️ Local Development Setup

To run NeoSight locally, you need to spin up both the Express backend and the Vite frontend.

### 1. Launch the Analytics Backend
The backend serves the document processing endpoint and handles simulated multipart file latency.

```bash
cd backend
npm install
node server.js
```
*(Note: The server is hardcoded to launch on Port `5001` to prevent 'EADDRINUSE' conflicts with macOS AirPlay Receiver).*

### 2. Launch the Officer Dashboard (Frontend)
The frontend houses the ultra-premium Landing Page and the 4-Step Forensics Pipeline.

```bash
cd frontend
npm install
npm run dev
```

Navigate to `http://localhost:5173/` in your browser.
